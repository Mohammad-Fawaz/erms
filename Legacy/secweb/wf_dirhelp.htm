ERMS Workflow Help

Introduction - The Workflow Process In a Nutshell

Workflow in the ERMS system allows users to quickly define, assign and create tasks to handle common processes. The workflow tools provided by the system allow for the creation of workflow templates which are utilized in the creation of tasks.

In order to provide a usable workflow system, certain steps should be taken to ensure that data is available for the creation of specific workflow items. The workflow process can be broken down as follows:

1. Workflow Group Creation and the Addition of Members

This should be your first step in creating a usable workflow system. Workflow Groups and their members form the basis for all task assignment within the workflow system. Some groups created could have many members, but every group must contain at least one member to ensure proper processing.

Workflow Groups and members are discussed in more detail in: About Workflow Groups and Creating Workflow Groups and Adding Members.

2. Action Template Creation

Action templates or Actions as they will be referred to throughout the rest of information contained here provide the ability to contain specific tasks which can then be used by any number of Workflow Templates. Specific actions may even be revisited multiple times within the process steps of a workflow template. Such a case might occur in the case of a review that would be conducted before other process steps occurred, and again at the time all other process steps have been completed. Actions should be thought of as generalizations that are further defined when they are added to workflow templates.

Workflow Actions are discussed in more detail in: About Workflow Actions, Creating Actions and Creating Templates and Adding Action Items.

3. Workflow Template Creation

Workflow Templates provide a container to pull all the pieces together into a coherent whole. Templates contain basic header information that identifies a template and it's intended use, and Template Items that contain action information but provide for a greater level of detail. These items, when added to a template, are no longer the generalized Actions they were based upon, but are their own containerized task definitions.

Workflow Templates and Template Items are discussed in greater detail in: About Workflow Templates and Items, Creating Templates and Adding Action Items and About Task Generation.

4. Workflow Task Generation

Workflow Tasks are generated based on information contained within a Workflow Template. These tasks exist within the ERMS Task system in order to allow the same reporting capabilities as standard tasks, but they contain additional information required for the completion of tasks within the confines of the step by step process defined by the Workflow Template. This allows these tasks to be activated at the proper time by the completion of previous steps in the process, or they may be reactivated (regenerated) or permanently deactivated if a rejection occurs or an approval is not satisfied.

A group of Workflow Tasks is generated from the designation of a base reference available within the ERMS system. When a document record is created within the ERMS system, Workflow Tasks can be generated to accomplish the production and release of the document. When a change is requested Workflow Tasks can be generated to accomplish the updates and fulfillment of the change.

Task generation is kicked off from the view of a specific item within the ERMSWeb system. This point of entry provides the base reference and allows a specific Workflow Template to be selected. Once selected the process steps contained within the template are dispalyed to allow for the inclusion or ommision of specific steps made available by the Template Items and a group of tasks is then created including the base reference information and provisions/directions of the process steps contained within the selected items. When a group of Workflow Tasks is generated for a base reference, these tasks will be displayed as a part of the viewable record for the base reference. This provides quick reference to progress through the process steps.

Workflow Tasks and Task Generation are discussed in greater detail in: About Task Generation, Generating Workflow Tasks, Adding and Modifying Workflow Tasks and Workflow Operations - Updating Workflow Tasks.

5. Workflow Operations

Workflow Operations refers to the work of updating and completing Workflow Tasks, in other words, accomplishing the tasks set forth by the workflow task records. From the users perspective this will be as simple as going through their To Do List and providing minimal information to the system. From this interface various operations will be accomplished, including approving/denying requests, accepting/rejecting reviewed materials, and providing time spent and/or cost information as well as completion of process tasks.

Workflow Operations are discussed in greater detail in: Workflow Operations - Updating Workflow Tasks.
